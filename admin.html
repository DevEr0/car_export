How this works — read once

Edit anything below, then choose how to save:

① Preview on this device — see your changes on the live site immediately, but only in this browser. Good for checking before going public.

② Publish — makes changes visible to everyone. Because this is a static GitHub Pages site, publishing means saving the data.js file back to your repository — either by downloading it and uploading to GitHub, or by committing directly with a GitHub token (set up once). Both are on the Publish tab.

Option A — Download & upload

Easiest. No setup. Works for anyone.

1Click Download data.js.
2Go to your repo on github.com → open data.js → click the ✏️ pencil.
3Select all, paste the new file, and Commit changes.
4GitHub Pages republishes in ~1 minute. Done — live for everyone.

Option B — Commit directly

One-time setup, then publishing is one click forever.

Create a token at github.com → Settings → Developer settings → Fine-grained tokens, scoped to this repo with Contents: Read & write. Anyone with this token can edit the repo, so keep it private.

🔒 Security — admin password

This password protects admin.html on your live site. The site's content (cars, text) is also protected separately — publishing always needs your GitHub token, so visitors can never change the real site even if they open the admin.

Changes take effect for everyone after you Publish. (You stay logged in on this device.)
