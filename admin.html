Option A — Download & upload方式 A —— 下载并上传

Easiest. No setup. Works for anyone.最简单，无需设置，人人可用。

1Click Download data.js.点击「下载 data.js」。
2Go to your repo on github.com → open data.js → click the ✏️ pencil.打开 github.com 上的仓库 → 打开 data.js → 点击 ✏️ 铅笔图标。
3Select all, paste the new file, and Commit changes.全选，粘贴新文件，然后点击 Commit changes。
4GitHub Pages republishes in ~1 minute. Done — live for everyone.GitHub Pages 约 1 分钟后重新发布。完成 —— 所有人可见。

Option B — Commit directly方式 B —— 直接提交

One-time setup, then publishing is one click forever.设置一次，之后一键发布。

Create a token at github.com → Settings → Developer settings → Fine-grained tokens, scoped to this repo with Contents: Read & write. Anyone with this token can edit the repo, so keep it private.在 github.com → Settings → Developer settings → Fine-grained tokens 中创建，仅授权本仓库的 Contents：读写 权限。持有此令牌者可编辑仓库，请妥善保管。

🔒 Security — admin password安全 —— 管理密码

This password protects admin.html on your live site. The site's content (cars, text) is also protected separately — publishing always needs your GitHub token, so visitors can never change the real site even if they open the admin.此密码用于保护您线上网站的 admin.html。网站内容（车辆、文字）另有保护 —— 发布始终需要您的 GitHub 令牌，因此即使访客打开后台，也无法更改真实网站。

Changes take effect for everyone after you Publish. (You stay logged in on this device.)改动在您「发布」后对所有人生效。（本设备保持登录状态。）
